BoardgameGeek Api

I'm a huge nerd, so naturally boardgames are one of my most enjoyable hobbies! Over time, I've frequently purchased games on kickstarter or at the local game store that I have found through the BoardgameGeek website. As my first Capstone, I set out to make a usable ui that would list the many various statistics for board games without overwhelming the user.

Being the sole developer on this project, I was responsible for developing the mvp (minimum viable product) in both concept and execution, as well as the polished version. This included the app architecture, styling, ajax calls, and code structure. After developing the mvp, I submitted the project for peer review and made changes to features as well as bug fixes.

Major challenges

The first major challenge was circumventing a CORS error due to BoardgameGeek Api not having their CORS headers enabled.

    var cors_api_host = // proxy for CORS here: 'corsproxy.' ;

    XMLHttpRequest.prototype.open = function() {
        var args = slice.call(arguments);
        var targetOrigin = /^https?:\/\/([^\/]+)/i.exec(args[1]);
        if (targetOrigin && targetOrigin[0].toLowerCase() !== origin &&
            targetOrigin[1] !== cors_api_host) {
            args[1] = cors_api_url + args[1];
        }
        return open.apply(this, args);
    };

The main difficulty in working with this API was that it was XML. The data needed to be converted to JSON so that it could be parsed and processed appropriately. 

        
          function xmlToJson(xml) {
	var obj = {};

	if (xml.nodeType == 1) {
		if (xml.attributes.length > 0) {
		obj["@attributes"] = {};
			for (var j = 0; j < xml.attributes.length; j++) {
				var attribute = xml.attributes.item(j);
				obj["@attributes"][attribute.nodeName] = attribute.nodeValue;
			}
		}
	} else if (xml.nodeType == 3) { // text
		obj = xml.nodeValue;
	}

	// do children
	if (xml.hasChildNodes()) {
		for(var i = 0; i < xml.childNodes.length; i++) {
			var item = xml.childNodes.item(i);
			var nodeName = item.nodeName;
			if (typeof(obj[nodeName]) == "undefined") {
				obj[nodeName] = xmlToJson(item);
			} else {
				if (typeof(obj[nodeName].push) == "undefined") {
					var old = obj[nodeName];
					obj[nodeName] = [];
					obj[nodeName].push(old);
				}
				obj[nodeName].push(xmlToJson(item));
			}
		}
	}
	return obj;
};

    Final Fantasy Quiz

    Growing up, I was a huge fan of rpgs, and in particular the creations of then Squaresoft (now SquareEnix). Particularly I loved the stories with incredibly well developed characters, mature themes, and strategic gameplay. If your opinion is that Final Fantasy VII is the best Final Fantasy, I will argue with you (it was good but the depth of Final Fantasy VI was unmatched)!

    With this project, my goal was to recreate the look and feel of the classic final fantasy menu and test user's knowledge about classic games in the series. I wanted this project to be a memorable experience, and with that in mind I wanted the user to be able to select a character.

    The beginning of the project involved writing out the user experience, and some notes about the inner logic of the application. The goal was to mock up the html of the landing page and main quiz page and wireframe basic styling. As my work on the project progressed, I worked on editing character sprites in Adobe Photoshop and sound effects in Adobe Audition.

    There were many challenges in this project including asynchronous javascript, custom radio button logic, appropriate use of event listeners, and implementation of styling and the interface.
